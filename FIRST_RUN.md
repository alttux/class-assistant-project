# 🎓 Classroom Manager - Первый запуск

## 1️⃣ Установка на сервер (машина учителя)

### Шаг 1: Откройте терминал и перейдите в директорию проекта

```bash
cd /Volumes/ADATA/PROGRAMMING/class-assistant-project/lab_manager
```

### Шаг 2: Установите зависимости

```bash
pip install -r requirements.txt
```

Ожидаемый результат:
```
Successfully installed fastapi uvicorn sqlalchemy pyqt6 psutil httpx pydantic reportlab python-multipart websockets
```

### Шаг 3: Запустите сервер

```bash
python -m server.gui.main_window
```

Вы должны увидеть окно приложения "🎓 Classroom Manager".

## 2️⃣ Установка на рабочие станции (студенты)

На каждом компьютере в классе:

### Шаг 1: Откройте терминал

### Шаг 2: Перейдите в директорию проекта

```bash
cd /Volumes/ADATA/PROGRAMMING/class-assistant-project/lab_manager
# На Windows: cd path\to\class-assistant-project\lab_manager
```

### Шаг 3: Установите зависимости

```bash
pip install -r requirements.txt
```

### Шаг 4: Запустите агент

```bash
python -m agent.main
```

Вы должны увидеть:
```
INFO:     Uvicorn running on http://0.0.0.0:8001 (Press CTRL+C to quit)
```

**⚠️ Важно**: Оставьте это окно открытым! Агент должен постоянно работать.

## 3️⃣ Обнаружение и управление

### На машине учителя:

1. Откройте GUI приложение (если еще не открыто)
2. Нажмите кнопку **▶️ Запустить сервер**
3. Перейдите в меню **⚙️ Управление** → **🔍 Поиск агентов**
4. Подождите 30-60 секунд

Когда агенты будут найдены:
- ✅ Статусная строка покажет количество найденных ПК
- ✅ В вкладке "🎮 Управление" появится список рабочих станций
- ✅ Вы сможете отправлять команды на любой ПК

### Управление рабочими станциями

1. Выберите нужный ПК из списка (или "Все ПК")
2. Выберите действие:
   - 🔒 **Заблокировать экран** - заблокировать рабочее место
   - 🛑 **Выключить ПК** - выключить компьютер
   - 🔄 **Перезагрузить** - перезагрузить систему
   - 🚫 **Блокировать интернет** - заблокировать доступ в интернет
   - ✅ **Разблокировать интернет** - вернуть интернет

3. Нажмите кнопку с действием
4. Дождитесь результата в статусной строке

## ⚡ Быстрая помощь

### Если сервер не запускается

```bash
# Проверьте, свободен ли порт 8000
lsof -i :8000

# Если порт занят, найдите процесс и завершите его
kill -9 <PID>
```

### Если агенты не найдены

```bash
# Проверьте, запущены ли агенты на каждом ПК
ps aux | grep agent.main

# Проверьте, открыт ли порт 8001
nc -zv 192.168.1.10 8001

# Проверьте брандмауэр на рабочей станции
# На macOS: System Preferences → Security & Privacy → Firewall
# На Windows: Windows Defender Firewall → Allow app through firewall
```

### Если приложение зависает

```bash
# Нажмите CTRL+C в терминале

# Перезапустите
python -m server.gui.main_window
```

## 🎯 Функциональность

### Доступные команды

- **lock_screen** - Заблокировать экран
- **unlock_screen** - Разблокировать экран (если реализовано)
- **shutdown** - Выключить ПК
- **restart** - Перезагрузить ПК
- **block_internet** - Блокировать интернет
- **unblock_internet** - Разблокировать интернет

### Скоро добавится

- Удаленное управление через интернет
- Запись видео с экранов студентов
- Распределенные сценарии уроков
- Экспорт отчетов в PDF

## 💡 Советы

1. **Запуск агентов автоматически**
   - Добавьте `python -m agent.main` в автозагрузку
   - На Windows: Task Scheduler
   - На macOS: LaunchAgent
   - На Linux: systemd service

2. **Улучшение производительности**
   - Используйте проводное соединение (Ethernet) вместо WiFi
   - Убедитесь, что все машины в одной подсети
   - Закройте ненужные приложения на ПК

3. **Безопасность**
   - Используйте HTTPS для сетевого соединения (в production)
   - Меняйте AUTH_TOKEN регулярно
   - Ограничьте доступ на уровне сети (firewall rules)
