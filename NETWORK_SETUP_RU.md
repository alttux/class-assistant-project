# Руководство по настройке соединения в локальной сети

## Проблема
Приложение работало только на `localhost` и не могло подключиться к агентам в локальной сети.

## Решение
Были внесены следующие изменения:

### 1. **Конфигурация сервера (constants.py)**
- Добавлена функция `get_local_ip()` для автоматического определения IP адреса машины в сети
- Добавлена переменная `SERVER_IP` для использования настоящего IP вместо localhost
- Сервер теперь слушает на `0.0.0.0` (все интерфейсы вместо localhost)

### 2. **API сервер (api.py, server/main.py)**
- Обновлены URL подключения к использованию `SERVER_IP` вместо `localhost`
- Сервер запускается с `host="0.0.0.0"` для приема соединений с любого интерфейса

### 3. **Конфиг агента (agent/config.json)**
- Добавлено поле `server_url` для указания адреса сервера в сети
- Добавлено поле `listen_address` с значением `0.0.0.0`

### 4. **GUI интерфейс**
- Обновлены все ссылки на серверные адреса с localhost на SERVER_IP
- Реализована функция автоматического обнаружения агентов в сети
- Добавлена возможность динамического обновления списка подключённых рабочих станций

### 5. **Обнаружение агентов (network_discovery.py)**
- Создан новый модуль для сканирования локальной сети
- Автоматически определяет диапазон сети на основе локального IP
- Сканирует порт 8001 для поиска активных агентов

## Как использовать

### Запуск сервера

```bash
# Перейти в директорию проекта
cd /Volumes/ADATA/PROGRAMMING/class-assistant-project/lab_manager

# Установить зависимости (если не установлены)
pip install -r requirements.txt

# Запустить GUI приложение (рекомендуется)
python -m server.gui.main_window

# ИЛИ запустить API сервер напрямую
python -m server.api
```

### Запуск агента на каждой рабочей станции

На каждом компьютере в классе выполнить:

```bash
# На Windows/Mac/Linux
python -m agent.main

# Или с использованием uvicorn напрямую
uvicorn agent.main:app --host 0.0.0.0 --port 8001
```

### Автоматическое обнаружение агентов

1. Убедитесь, что все агенты запущены на рабочих станциях
2. В GUI приложении перейдите в меню "⚙�� Управление" → "🔍 Поиск агентов"
3. Приложение автоматически найдет все агенты в локальной сети
4. Список рабочих станций обновится автоматически

### Ручное указание агентов

Если автоматическое обнаружение не работает, укажите IP адреса вручную:

1. Найдите IP адреса всех рабочих станций (ping hostname или ipconfig)
2. В файле `server/gui/control_panel.py` обновите словарь `workstations_ips`
3. Перезапустите приложение

## Диагностика проблем

### Агенты не обнаруживаются

1. **Проверьте соединение**:
   ```bash
   ping <agent_ip>
   # Должен быть ответ
   ```

2. **Проверьте, запущены ли агенты**:
   ```bash
   nc -zv <agent_ip> 8001
   # Должно быть: Connection to <agent_ip> port 8001 [tcp/*] succeeded!
   ```

3. **Проверьте брандмауэр**:
   - Убедитесь, что порты 8000 и 8001 открыты в брандмауэре
   - Отключите брандмауэр на время тестирования

### Медленное обнаружение

- По умолчанию сканирование ждет 2 секунды для каждого IP
- Скорость зависит от размера подсети и количества активных машин
- Для больших сетей рекомендуется указать MAC адреса известных машин

## Архитектура соединения

```
Учитель (GUI) на сервере
    ↓ (HTTP на 0.0.0.0:8000)
FastAPI сервер (слушает на всех интерфейсах)
    ↓ (HTTP на 0.0.0.0:8001 для каждого агента)
Агенты на рабочих станциях
    ↑ (ответы и статус)
```

## Ключевые изменения в коде

1. **constants.py**: Добавлены функции для определения локального IP
2. **api.py**: Использует SERVER_IP вместо localhost
3. **server/main.py**: Обновлено подключение API клиента
4. **agent/config.json**: Может указывать server_url
5. **gui/main_window.py**: Реализовано обнаружение агентов
6. **gui/control_panel.py**: Динамическое обновление списка рабочих станций
7. **network_discovery.py**: Новый модуль для сканирования сети

## Безопасность

⚠️ **Важно**: Текущая конфигурация использует простой токен для аутентификации. Для production окружения рекомендуется:

1. Использовать HTTPS вместо HTTP
2. Применить более сильную аутентификацию (OAuth, JWT)
3. Добавить шифрование данных в полёте
4. Ограничить доступ на уровне сети (VPN, IP whitelist)
5. Добавить логирование и мониторинг всех соединений
